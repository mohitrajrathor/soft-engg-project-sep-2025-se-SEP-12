  The Problem

  The error occurred because:
  1. ConversationBufferMemory was imported inside a try-except block
  2. It was used in type hints (line 26 and 139) which are evaluated at class definition time
  3. When Python tried to evaluate the type hints, ConversationBufferMemory wasn't available in the global namespace

  The Fix

  I updated the imports to use Python's TYPE_CHECKING pattern:

  from typing import Dict, Optional, AsyncIterator, Any, TYPE_CHECKING

  if TYPE_CHECKING:
      # Imports for type checkers only (mypy, IDE)
      from langchain_google_genai import ChatGoogleGenerativeAI
      from langchain.memory import ConversationBufferMemory
      from langchain.chains import ConversationChain

  try:
      # Actual runtime imports
      from langchain_google_genai import ChatGoogleGenerativeAI
      from langchain.memory import ConversationBufferMemory
      from langchain.chains import ConversationChain
      LANGCHAIN_AVAILABLE = True
  except ImportError:
      LANGCHAIN_AVAILABLE = False
      # Fallback when LangChain is not available
      ChatGoogleGenerativeAI = None
      ConversationBufferMemory = None
      ConversationChain = None
